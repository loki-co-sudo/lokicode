# ディープシンクのモデル要件ゲート — 設計仕様

対象バージョン: 1.3.0
関連: [deep-reasoning-v2.md](deep-reasoning-v2.md) / [agent-v3.md](agent-v3.md)（#2 強い検証器）

## 1. 動機

ディープシンクが単一の Agent モードより**精度で勝てる**のは、次の2条件が成り立つときだけ:

1. **検証器（JUDGE）・ブリーフ・最終合成を担う合成モデルが「賢さの壁」の上**にあること。
   過去の実測で、ディープシンクが Agent 単独に負けた主因は「思考/検証が無料ルーター
   （壁の下）」だったこと（agent-v3 #2 の動機）。弱い検証器は誤りを見逃すため、
   多段パイプラインのコストだけ払って品質が上がらない。
2. **思考モデルがツール（読み取り）を使えて調査を接地できる**こと。調査フェーズが
   接地できないと、証拠が空か捏造になり、以後の全フェーズが汚染される。

現状の警告（synthWeakReason）は合成モデルの「ルーター系/変動価格/ツール非対応」しか
見ておらず、**知能の下限**と**思考モデル側**を判定していない。

## 2. 必要知能の判断基準（このリポジトリの公式見解）

モデル一覧には Artificial Analysis の知能指数（`intelligenceIndex`）が来る（未公開なら null）。
指数のおおまかな目安: 8B級（MMLU 65% 前後＝「賢さの壁」）≈ 20 台前半、
中堅（Gemini Flash / DeepSeek V3 級）≈ 40 台、フロンティア ≈ 60 以上。

| 役割 | 最低ライン（下回ると critical） | 推奨ライン（下回ると warn） | 根拠 |
| --- | --- | --- | --- |
| 合成（検証器・ブリーフ・最終） | 知能指数 **35** | 知能指数 **45** | 検証器は他モデルの微妙な誤りを検出する側。壁(≈20台)より十分上の中堅級が最低条件。45 未満は「見逃しが増える」灰色地帯 |
| 思考（調査・ドラフト・実行） | ツール対応必須 | 知能指数 **25** | 調査は読解と構造化が主業務で壁ぎりぎりでも回る（検証器が誤りを拾う前提）。ただしツールを使えないと接地不能で Agent 単独より劣化する |

指数が null（未公開）のモデルは指数では判定しない（誤検知より見逃しを許容）。

## 3. 判定ルール（lib/modelGate.ts・純ロジック）

`assessDeepThinkReadiness(thinkingId, synthesisId, models)` → `{ level, issues[] }`

- 合成モデル:
  - ルーター系メタモデル（openrouter/free・auto・fusion・pareto-code）→ **critical**
    （実際に当たるモデルが不明＝検証器の賢さを保証できない）
  - 変動価格（price < 0）→ **critical**
  - 知能指数 < 35 → **critical** ／ 35–44 → **warn**
  - ツール非対応 → **warn**（最終の事実再確認と実行計画の質が低下）
- 思考モデル:
  - ルーター系 → **warn**（ツール対応・品質が引き次第。安価な固定モデルを推奨）
  - ツール非対応 → **critical**（調査・実行フェーズが接地不能＝Agent 単独より精度が落ちる）
  - 知能指数 < 25 → **warn**
- モデル一覧に無い ID・一覧が空（オフライン）→ 判定しない（issues なし）。
- level は issues の最大深刻度（critical > warn > ok）。

## 4. UI

- ChatPane の既存 synthWeakReason バナーを置換。ディープシンク ON（または「オート」ON＝
  deep に振り分けられうる）のとき、warn は琥珀色・critical は赤で全 issue を列挙し、
  「なぜダメか＋どうすればよいか」を1行ずつ表示。設定へのリンクを維持。
- ブロックはしない（警告のみ）。ユーザーが承知で回す自由を残す。

## 5. 検証

- `npm test`: ルーター/低知能/ツール非対応/未知 ID/一覧空 の各分岐。
- 手動: 合成に無料ルーターを設定→赤バナー、Gemini 2.5 Pro 等→バナー消灯。
