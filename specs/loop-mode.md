# ループモード（検証ループの強化）— 設計仕様

対象バージョン: 1.2.0
関連: [effort-presets.md](effort-presets.md) / [agent-v3.md](agent-v3.md)（#3 実行接地の自己修正）

## 1. 動機と導入判断

「変更→検証→修正を、チェックが通るまで自動で回す」ループ開発は、Reflexion 系の
実行接地自己修正そのもので、本 repo には既に萌芽がある（設定「検証コマンド」＋
最大2回の自動修正、0.43.0）。よって導入自体には賛成。ただし次の実リスクがあるため、
**既定 OFF のオプトイン・モード**として組み込む:

| リスク | 対策 |
| --- | --- |
| コスト暴走（失敗のたびに Agent 実行1周ぶんの API 呼び出し） | 上限5回で打ち切り。既定 OFF・トグルのツールチップにコスト増を明記 |
| 同じ誤りの堂々巡り（思考ロック） | **同一エラー2回連続**で即停止し、別コンテキスト（新スレッド/強モデル）での修復を促す |
| テストの弱体化で「通す」不正 | システム指示と修正プロンプトの両方で禁止を明文化（直すのはコード、評価基準ではない） |
| 証拠なしの「完了」報告 | 成功時は**通過した出力の末尾を証拠として**チャットに添付。失敗打ち切り時は残状況を報告 |
| フレーキー（不安定）なテスト | 同一エラー検知が擬似的に効く（揺れる出力は署名が変わる＝5回上限側で停止） |

## 2. 仕様

### 2.1 設定
- トグル「**ループモード**」（`lokicode.loopMode`、既定 OFF）。Agent モード（非ディープ）
  で表示。検証コマンド（`lokicode.verifyCommand`）が空のときは ON でも走らない
  （送信時にその旨を1回案内）。

### 2.2 ループ本体（ChatPane の検証ループを拡張）
1. 【変更】通常の Agent 実行がファイルを編集。
2. 【検証】検証コマンドを自動実行（例 `npm test` / `npm run build; npm test`）。
3. 【修正】exit code ≠ 0 なら、エラーログを渡して Agent に修正させ 2 に戻る。
- 【成功】exit code = 0 で終了。**出力末尾（最大12行）を証拠として添付**して完了報告。
- 【上限】ループモード ON は最大 **5** 回（`LOOP_MAX_ATTEMPTS`）。OFF は従来どおり最大2回。
  使い切ったら停止し「何が残っているか」をログ付きで報告。
- 【手詰まり】エラーログの**正規化署名**（数値→#・空白圧縮・末尾600字）が2回連続で一致
  したら思考ロックと判断して即停止し、新スレッド（別コンテキスト）や強モデルへの切替を促す。

### 2.3 プロンプト（ループモード ON のとき追記）
- テスト/アサーションの削除・弱体化・スキップで通そうとすることを禁止。
- チェック通過の証拠なしに「完了」と主張しないこと（証拠はハーネスが添付する）。

### 2.4 実装配置
- 純ロジック（上限値・エラー署名）: `src/lib/loop.ts`（+ `loop.test.ts`）。
- ループ制御: `ChatPane.tsx` の既存検証ループを置換。
- 深いパイプライン（ディープシンクの実行フェーズ）への適用は将来枠
  （実行フェーズは onIdle 継続ガードが既にあるため、まず Agent モードで実証する）。

## 3. 検証方法
- `npm test`: errorSignature の同値/差分判定。
- 手動: 検証コマンドに `npm test` を設定→わざとテストを壊す依頼→ループが修正し、
  ✅ 報告に出力末尾が引用されること。同じ壊れ方を直せないケースで手詰まり停止が出ること。
